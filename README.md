## Prerequisites

- [Node.js](https://nodejs.org/) **v20.9.0 or higher** (see `engines` in `package.json`)
- [npm](https://www.npmjs.com/) (bundled with Node.js)

<!-- GETTING STARTED -->
## Getting Started

1. Clone the repository:
   ```bash
   git clone https://github.com/HiveSolution/TypeScript-ESLint-Quickstart.git
   cd TypeScript-ESLint-Quickstart
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Start coding – your entry point is **`src/index.ts`**.

<!-- AVAILABLE SCRIPTS -->
## Available Scripts

| Command | Description |
|---|---|
| `npm run dev` | Run the project in **development mode** with hot-reload via `ts-node-dev`. Changes to source files are automatically picked up without a manual restart. |
| `npm run build` | Lint the source files and then compile TypeScript to JavaScript in `dist/`. |
| `npm start` | Run the compiled JavaScript output (`dist/index.js`). Run `npm run build` first. |
| `npm run lint` | Check all TypeScript source files in `src/` for ESLint rule violations. |
| `npm run lint:fix` | Automatically fix any auto-fixable ESLint (and Prettier) issues in `src/`. |

### Development workflow

```bash
# Start the dev server with hot-reload
npm run dev

# Check for lint errors
npm run lint

# Auto-fix lint / formatting errors
npm run lint:fix

# Build a production-ready dist/
npm run build

# Run the compiled output
npm start
```

<!-- CONFIGURATION -->
## Configuration

### TypeScript (`tsconfig.json`)

- **Target**: `es2020`
- **Module**: `commonjs`
- **Strict mode**: enabled
- **Output directory**: `dist/`
- **Root directory**: `src/`

### ESLint (`eslint.config.mjs`)

Uses the [flat config](https://eslint.org/docs/latest/use/configure/configuration-files-new) format and bundles:

- [`@eslint/js`](https://www.npmjs.com/package/@eslint/js) – recommended JavaScript rules
- [`typescript-eslint`](https://typescript-eslint.io/) – TypeScript-aware rules
- [`eslint-plugin-prettier`](https://github.com/prettier/eslint-plugin-prettier) – Prettier formatting enforced as ESLint errors

Notable rules:
- `@typescript-eslint/explicit-function-return-type` – warn when return types are omitted
- `@typescript-eslint/naming-convention` – enforce naming for private members (properties: camelCase with leading `_`; methods: camelCase without leading `_`)
- `indent` – 2-space indentation
- `prefer-const` – warn when `let` can be `const`

### Prettier (`.prettierrc.json`)

| Option | Value |
|---|---|
| `singleQuote` | `true` |
| `trailingComma` | `none` |
| `tabWidth` | `2` |
| `semi` | `true` |
| `printWidth` | `100` |
